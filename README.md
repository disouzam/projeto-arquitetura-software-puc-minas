# Propósito

Esse repositório contém algumas atividades executadas para a disciplina Projeto de Arquitetura de Software, do currículo do curso de [Arquitetura de Software Distribuído](https://vemprapuc.pucminas.br/arquitetura-de-software-distribuido-online), lecionada pelo professor [Marco Mendes](https://github.com/marco-mendes), oferta 9, turma 1, 2025/2026.

O conteúdo presente aqui é fruto da minha escrita individual, suportada por material educacional disponível na internet (com referências mencionadas sempre que necessário) e também faz uso de modelos de linguagem natural disponíveis (o prompt e o conteúdo na íntegra será disponibilizado em arquivos separados)

# Architectural Decision Records

Os documentos chamados de `Architectural Decision Records` (Registros de decisão arquitetural, em livre tradução) foram propostos por Michael Nygard (conforme citado por Marco Mendes em [Registros de Decisões Arquiteturais (ADR)](https://github.com/marco-mendes/projeto-arquitetura-software/blob/main/2.1%20ADR.md)) como mecanismo de formalizar as decisões de arquitetura de software bem como prover registro histórico do contexto em que a decisão foi tomada, incluindo as restrições existentes, a capacitação técnica do time e outras limitações existentes. 

Nesse repositório serão, inicialmente, escritas duas ADRs, como são commumente referidos esses tipos de documentos, como parte dos requisitos para aprovação na discipline Projeto de Arquitetura de Sofware:

- ADR-003 – Estratégia de Identidade, Autenticação e Autorização, incluindo SSO
- ADR-004 – Estratégia de Governança, Proteção e Auditoria de Dados
